# Домашнее задание к занятию «Репликация и масштабирование. Часть 2» Дедяхин Игорь

### Задание 1

Опишите основные преимущества использования масштабирования методами:

- активный master-сервер и пассивный репликационный slave-сервер; 
- master-сервер и несколько slave-серверов;


*Дайте ответ в свободной форме.*

### Решение

Первый вариант (активный master-сервер и пассивный репликационный slave-сервер) повышает надежность (если основной сервер выходит из строя, есть возможность быстрого переключения на реплику) и снижает нагрузку на основной сервер, перенаправляя операции чтения на slave.
Второй вариант (master-сервер и несколько slave-серверов) позволяет распределить все запросы на чтение между несколькими серверами, увеличивая производителность системы.

### Задание 2


Разработайте план для выполнения горизонтального и вертикального шаринга базы данных. База данных состоит из трёх таблиц: 

- пользователи, 
- книги, 
- магазины (столбцы произвольно). 

Опишите принципы построения системы и их разграничение или разбивку между базами данных.

*Пришлите блоксхему, где и что будет располагаться. Опишите, в каких режимах будут работать сервера.* 

### Решение


Вертикальный шардинг: разделить БД на три сервера. 
Первый сервер содержит таблицу "Пользователи". Здесь лучше использовать режим master-сервер и несколько slave-серверов. Данные пользователей это критически важная информация, слеовательно нужна высокая отказоустойчивость. Запись данных о новых пользователях будут идти на master, а чтение будет распределяться междe slave. Если (если master выходит из строя, есть возможность быстрого переключения на реплику).
Второй сервер содержит таблицу "книги". Здесь лучше так же лучше использовать режим master-сервер и несколько slave-серверов, т.к. будет юольшая нагружка на чтение. Запись данных о новых книгах будут идти на master.
Третий сервер содержит таблицу "магазины". Здесь дотсаточно master и один slave. Открытие новых магазинов и закрытие старых - события нечастые и данные о них будут идти на master, slave будет использоваться для чтения и в качестве резерва.
